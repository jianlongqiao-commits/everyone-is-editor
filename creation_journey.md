## 第一幕：起点 - 一个提示词的困惑

### 用户的初始问题

用户提供了一个长达数千字的"去AI味"写作提示词，核心特点：
- 角色设定：资深自媒体专栏作家
- 任务：根据标题生成1200字左右的公众号文章
- 重点：大量"禁止事项"来去除AI味
  - 禁用逻辑连接词（首先、其次、然后、总之）
  - 禁用序号（一二三、123）
  - 段落结尾无标点
  - 避免总-分-总结构

**用户的第一个问题**：
> 如果想通过"腾讯朱雀检测系统"这个可以吗？

## 第二幕：现实检验

### 我的分析：不能保证100%通过

**核心观点**：
1. **朱雀检测的原理不只看表面**
   - 深层语义模式
   - 词汇分布的统计规律
   - AI特有的"指纹"

2. **这个提示词的局限**
   - ✅ 能显著降低"AI味"
   - ✅ 可能骗过简单检测工具
   - ❌ 面对朱雀这种企业级系统，通过率不确定
   - ❌ 随着时间推移，效果会递减

3. **更稳妥的做法**
   - 用AI生成初稿 → 自己大幅改写
   - 混合创作：框架用AI，核心观点自己写
   - 多轮润色：打破AI模式
   - 加入真实细节：AI编不出的具体数据、亲身经历

**关键洞察**：
> 最关键的是：**把AI当助手而非替代品**

## 第四幕：关键转折点 - 融合卡兹克方法论

### 用户提供了新素材

用户给了一篇知名博主"数字生命卡兹克"的写作方法论，包含10条核心原则：

**心法层（最重要）**：
1. **活人感，是AI时代最贵的奢侈品**
   - AI可以模仿文笔，但没有体温、心跳、回忆、伤疤
   - 活人感 = 情感、偏见、成长、笨拙的自然流露

2. **真诚，是永远的必杀技**
   - 可以不写，但绝不欺骗
   - 一次夸大和隐瞒，换来短暂数据，失去长期信任

3. **价值观，比流量更重要**
   - 不是所有流量都要追
   - 长期品牌价值 > 短期流量数据

**战略层**：
4. **选题，决定了文章80%的生死**
   - 好选题 = 专业领域 + 读者兴趣 + 时间节点

5. **你必须先是一个专家**
   - 在具体问题上，研究和思考超过90%的读者

6. **不懂的不要硬写；要写的就往死里研究**

**技法层**：
7. **内容是故事，不是论文**
   - 用故事思维重新包装一切内容
   - 冲突 → 探索 → 发现 → 高潮 → 结局

8. **把读者当成：很聪明、很有钱、但很忙的人**
   - 不讲废话
   - 对得起读者的机会成本
   - 结构清晰，重点加粗

9. **HKR原则检查每个作品**
   - H（Hook/Happiness）：有趣吗？能勾住人吗？
   - K（Knowledge）：有料吗？能学到东西吗？
   - R（Resonance）：能产生共鸣吗？
   - 优秀内容：三者兼备 / 及格内容：至少两样

**修炼层**：
10. **评论区，是下一篇爆款的起点**
    - 读每一条评论
    - 做数据分析
    - 主观反馈 + 客观反馈

## 第五幕：深度洞察 - 用户的核心痛点

### 用户的关键反馈（转折点1）

> 我觉得以上适合检测一篇文章是否符合活人感\降低AI味道，但是**很多时候我会卡在写作这个环节**。

> 能通过**引导的方式**，补上这个环节吗？我给个意图，你来引导\拆解，并且引导的时候，就要考虑：
> 1. 这个选题是否有价值\是否有人看
> 2. 引导我说出一些**事实\经历\观点**，这是我独有的东西
> 3. 这样的内容才有可能吸引人看\并且去AI味道

**核心洞察**：
> 现有提示词的问题：它是个"执行器"，默认你已经有了素材和想法
> 真正需要的：一个**启发式引导流程**

我提出了三个方案：
1. **方案1：弹性字数区间** - 让话题复杂度决定字数
2. **方案2：用信息密度替代字数** ⭐️（用户最认可）
3. **方案3：分层写作目标** - 根据话题给建议

---

### 用户的第二个关键洞察（转折点2）

关于字数限制的深度讨论：

**用户的问题**：
> 为什么是1200字左右？如果不限制会怎么样？

**我的分析**：
- 1200字是基于快消内容的经验，不适合深度内容
- 硬性限制的坏处：
  - 话题展开到一半就得收尾
  - 为了字数而删减重要细节
  - 明明还有话要说，但被憋着
  - 把"凑字数"当成写作目标（本末倒置）

**更好的方案**：
- 轻度话题：800-1500字
- 中度话题：1500-2500字
- 深度话题：2500-5000字
- **质量优先于字数**

**用户的反馈**：
> 我觉得你说的这个特好：**方案2：用信息密度替代字数**。重点按照这个方向调整吧。

---

### 用户的第三个关键想法（转折点3）

> 我有个新的思路，可以融合进来：
>
> 就是我们**一起打磨出来一个大纲**，并且我确认之后，生成一个MD文档，**围绕这个文档一段段写**。
>
> 写一段，需要引入一个机制：**引入一个批判的角色**，跟我一起挑错\打磨，要保证真实性\吸引人的程度。

**我的反馈**：绝了！这个思路太棒了！

**为什么这个机制好**：
1. **分段写作 > 一次性生成**
   - 传统：一次性输出全文，用户只能事后修改
   - 新机制：逐段写作，每段都有参与和确认

2. **即时反馈 > 事后检查**
   - 传统：写完全文再检查，发现问题要大改
   - 新机制：每段即时打磨，问题不过夜

3. **批判角色 = 质量守门员**
   - 引入"批判视角"，专门挑刺
   - 确保每段都经得起推敲

4. **MD文档 = 可视化进度**
   - 随时知道写到哪了
   - 整体结构一目了然

**完整工作流程设计**：
```
阶段一：选题孵化（引导模式）
  ↓ 收集真实素材

阶段二：大纲打磨
  ├─ AI生成初稿大纲
  ├─ 用户确认/修改
  ├─ 生成MD文档（带TODO标记）
  └─ 确定最终大纲

阶段三：分段创作（循环）
  ├─ 写第N段
  ├─ 批判角色审查（4个维度）
  │   ├─ 真实性：细节经得起推敲吗？
  │   ├─ 吸引力：能抓住人吗？
  │   ├─ 信息密度：有废话吗？
  │   └─ AI味检测：太完美？太套路？
  ├─ 给出修改建议
  ├─ 用户确认/修改
  └─ 继续下一段

阶段四：整体审查
  ├─ HKR质量检验
  ├─ 全文信息密度检查
  └─ 最终优化
```

**批判角色的4个维度**：
1. **真实性拷问**：细节经得起推敲吗？有没有AI编的？
2. **吸引力检验**：读者会不会走神？有没有"钩子"？
3. **信息密度扫描**：在推进叙事吗？有没有废话？
4. **AI味检测**：句式太工整？有没有套路？

### 我提取的用户语言风格DNA

**1. 核心人设**：
- 懂技术、有审美、爱折腾的"实战派"观察者
- 不装专家，平视的"探索搭子"
- 敢于暴露情绪、有观点、不和稀泥

**2. 高频词汇**：
- **情绪词**：卧槽、讲真、TMD、说实话
- **网络梗**：牛马、赛博、搭子、实操、落地、贤者时间、斯国一、棺材板
- **中英夹杂**：Agent、Prompt、Vibe、All in one、MVP、OS

**3. 句式习惯**：
- **短句为主**："工具牛逼，还得方法跟上"
- **自问自答**："结果呢？它们只是静静地躺在那……"
- **生动比喻**："和我的健身房年卡一个下场"、"数字坟场"、"从屎里扒拉金子"

**4. 段落风格**：
- **极简主义**：很多段落只有1-2句话
- **手机友好**：避免大段文字块

**5. 开场三板斧**：
- **场景切入**："人在硅谷期间，我捯饬AI遇到问题..."
- **新闻引入**："山姆奥特曼坏事做尽，完全不考虑中国媒体老师的国庆假期..."
- **吐槽开场**："这两天朋友给我一个内测链接..."

**6. 结尾方式**：
- **反问收尾**："所以，看完文章，你vibe了吗？"
- **金句/升华**："看着另一个「牛马」在替你负重前行，感觉……还真不赖"
- **期待/展望**："作为一个普通用户，我无比期待重新找回初见AI的那种激动"

**7. 内容特点**：
- **实操至上**：必有prompt/工作流/工具推荐
- **有图有真相**：关键步骤配截图
- **犀利不留情**：敢吐槽大厂、质疑热点
- **共情感**：总是从用户/打工人视角出发

## 第七幕：大功告成 - 最终的Skill

### 创建的文件

```
~/.claude/skills/wechat-writer/
├── skill.md              # 主文件（18KB）
├── skill.json            # 配置文件
├── style_persona.md      # 用户的个人语言风格DNA
└── README.md             # 使用说明（10KB）
```

### 核心功能设计

**阶段一：选题孵化（9问引导）**

**第一轮：验证选题价值**
- 问题1：你为什么能写这个？（专业性）
- 问题2：读者为什么要看？（需求）
- 问题3：现在为什么是好时机？（时效性）

**第二轮：挖掘独特素材**（最重要）
- 问题4：你有什么别人没有的经历？
- 问题5：你有什么与众不同的看法？
- 问题6：你有什么独家的证据？

**第三轮：构建叙事框架**
- 问题7：这篇文章到底要说什么？（核心主张）
- 问题8：你打算怎么抓住读者？（开场设计）
- 问题9：你的文章要怎么展开？（故事线）

**阶段二：生成大纲**
- 基于真实素材生成结构化大纲
- 用户确认后创建MD工作文档

**阶段三：分段创作 + 批判审查**

每写完一段，立即从4个维度审查：

1. **真实性拷问**
   - 细节经得起推敲吗？
   - 有没有AI编的内容？

2. **吸引力检验**
   - 读者会不会走神？
   - 有没有"钩子"？

3. **信息密度扫描**（核心）
   - 这段在推进叙事吗？
   - 有没有"正确的废话"？
   - 每100字是否都在传递新信息？

4. **AI味检测**
   - 句式是否太工整？
   - 有没有套路？
   - 有没有"不完美"的地方？

**阶段四：全文审查**
- HKR质量检验（Hook、Knowledge、Resonance）
- 全文信息密度检查
- 最终优化

### 核心原则

1. **信息密度 > 字数**
   - 不设硬性字数限制
   - 质量标准：每句话都在传递价值
   - 素材丰富自然写长，素材简单自然写短

2. **真实素材 > 文笔技巧**
   - AI可以模仿文笔，但编不出真实经历
   - 用户提供的经历/观点/数据，才是护城河

3. **个人风格 > 完美表达**
   - 必须融入用户的语言习惯
   - 允许"不完美"（轻微重复/口语化/思路跳跃）
   - 这些"毛刺"才是真人写作的标志

4. **批判审查 > 一次性生成**
   - 分段写作，每段都有批判审查
   - 即时发现问题，即时修改

> 不是"AI写作工具"，而是"AI写作搭子"

**从**：
- 1200字

- 段落结尾无标点

- 信息密度（每句话都在传递价值）

## 方法论沉淀

### 如何打磨一个AI工具/Skill？

**第一步：理解真实需求**
- 不要满足于表面需求（"去AI味"）
- 深挖痛点（"卡在写作环节"）
- 找到本质问题（"缺少真实素材 + 引导流程"）

**第二步：学习最佳实践**
- 研究行业专家的方法论（卡兹克的10条原则）
- 提取可复用的框架（HKR检验、选题三要素）
- 融合到自己的工具中

**第三步：个性化定制**
- 学习用户的真实风格（读5篇文章）
- 提取语言特征（高频词汇、句式、段落风格）
- 内置到工具中（不是通用模板）

**第四步：设计工作流程**
- 不要只关注"输出"
- 设计完整的"输入 → 处理 → 输出 → 反馈"循环
- 引导式 > 命令式
- 分段打磨 > 一次生成

**第五步：质量机制**
- 不要依赖AI自检（容易自嗨）
- 引入"批判角色"
- 多维度审查（真实性、吸引力、信息密度、AI味）
- 即时反馈，即时修改

**第六步：迭代优化**
- 用户的每个反馈都是改进点
- 从"1200字限制"到"信息密度优先"
- 从"一次生成"到"分段打磨"
- 从"通用模板"到"个性化风格"
