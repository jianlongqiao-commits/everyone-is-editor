# 人人都是自己的主编

> 去AI味，写活人文章

一个通过引导式对话帮助你创作有价值、有温度、有个人风格文章的 Claude Code 技能。

## 核心特性

### 1. 不是"一键生成"，而是"协作创作"
- ❌ 传统AI写作：输入标题 → AI吐出全文 → 你事后修改
- ✅ 这个Skill：引导对话 → 挖掘真实素材 → 分段打磨 → 批判审查

### 2. 解决的核心问题
- **写不出来**：通过9问引导，帮你从模糊想法到清晰框架
- **AI味太重**：批判审查机制 + 个人风格融入，确保每段都"像人写的"
- **素材不够**：深挖你的独特经历/观点/数据，这是AI编不出来的护城河
- **质量不稳**：信息密度控制 + HKR检验，拒绝废话和注水

### 3. 四大工作模式

```
阶段一：选题孵化（引导模式）
  └─ 3轮9问：验证选题 → 挖掘素材 → 构建框架

阶段二：生成大纲
  └─ 基于真实素材生成结构化大纲 → 用户确认

阶段三：分段创作
  └─ 写一段 → 批判审查（4个维度）→ 用户确认 → 下一段

阶段四：全文审查
  └─ HKR质量检验 + 信息密度扫描 + 最终优化
```

### 使用方式

#### 场景1：我只有一个标题/想法

```
你：我想写一篇关于"AI工具如何提升工作效率"的文章

Skill会进入【引导模式】：
- 问你为什么能写这个话题（专业性）
- 问你读者为什么要看（需求）
- 问你现在为什么是好时机（时效性）

```

#### 场景2：我已经有素材和大纲

```
你：我想写XX话题，我的素材是：[你的经历/观点/数据]

Skill会进入【创作模式】：
- 直接生成大纲或开始分段写作
- 每段都会有批判审查
- 确保融入你的语言风格
```

## 核心机制详解

### 1. 选题孵化（9问引导）

**第一轮：验证选题价值**
- 问题1：你为什么能写这个？（专业性）
- 问题2：读者为什么要看？（需求）
- 问题3：现在为什么是好时机？（时效性）

**第二轮：挖掘独特素材**（最重要！）
- 问题4：你有什么别人没有的经历？
- 问题5：你有什么与众不同的看法？
- 问题6：你有什么独家的证据？

**第三轮：构建叙事框架**
- 问题7：这篇文章到底要说什么？（核心主张）
- 问题8：你打算怎么抓住读者？（开场设计）
- 问题9：你的文章要怎么展开？（故事线）

### 2. 批判审查机制

每写完一段，立即从4个维度审查：

**① 真实性拷问**
- 细节经得起推敲吗？
- 有没有AI编造的内容？
- 数据/案例是真实的还是瞎编的？

**② 吸引力检验**
- 读者会不会走神？
- 有没有"钩子"？（有趣的细节/意外的转折）
- 能让人继续往下读吗？

**③ 信息密度扫描**（核心）
- 这段在推进叙事吗？还是在原地打转？
- 有没有"正确的废话"？
- 如果删掉某句话，文章会不会有损失？

**④ AI味检测**
- 句式是否太工整？
- 有没有AI常用套路？
- 情绪是否真实？
- 有没有"不完美"的地方？（真人写作特征）

### 3. 个人风格融入

Skill会学习你的语言风格（基于`style_persona.md`），确保每段都带有你的个人特征：

**你的高频词汇**：
- 情绪词：卧槽、讲真、TMD
- 网络梗：牛马、赛博、搭子、斯国一
- 中英夹杂：Agent、Prompt、Vibe

**你的句式习惯**：
- 短句为主
- 自问自答
- 生动比喻

**你的段落风格**：
- 极简主义（很多段落只有1-2句话）
- 手机友好

### 4. 信息密度 > 字数

**不设硬性字数限制**，而是用质量标准：
- ✅ 每句话都在传递价值
- ✅ 没有可以无损删除的句子
- ✅ 每100-200字有一个"有意思"的点

**自动长度调节**：
- 素材丰富 → 自然写长（2000-3000字）
- 素材简单 → 自然写短（800-1500字）
- 关键是"把话说完"，而不是"凑字数"

## 使用技巧

### 1. 在引导阶段要诚实、具体

❌ 不要：
```
你：我对AI很感兴趣，想写这个话题
```

✅ 要：
```
你：我最近测试了5个AI写作工具，发现了一个有意思的现象...
（具体的时间、具体的工具、具体的发现）
```

### 2. 提供真实的素材

- **真实经历**：具体的时间、地点、对话、感受
- **真实观点**：你真正的想法，哪怕主观或偏激
- **真实数据**：你亲自测试的结果，不要编造

AI可以模仿文笔，但编不出真实的经历。你提供的素材，才是文章的护城河。

### 3. 在批判审查阶段要较真

看到审查报告后：
- 如果有"真实性问题"→ 补充具体细节
- 如果有"信息密度低"→ 狠心删掉废话
- 如果有"AI味过重"→ 改得更口语化、更随意

### 4. 善用MD文档跟踪进度

Skill会创建一个MD工作文档，记录：
- ✅ 哪些段落已完成
- 🔄 哪些段落正在写
- ⏳ 哪些段落待写作
- 📝 每段的批判审查记录

随时知道进度，文章结构一目了然。

## 常见问题

### Q1：这个Skill和直接让AI写文章有什么区别？

**传统AI写作**：
- 你输入标题 → AI一次性生成全文
- 内容都是AI"编"的（没有真实素材）
- AI味重，套路化
- 你只能事后修改，参与感弱

**这个Skill**：
- 引导你挖掘真实素材（经历/观点/数据）
- 分段写作，每段都有你的参与和确认
- 批判审查确保质量
- 融入你的个人语言风格
- 真正的"协作创作"

### Q2：我必须回答所有9个问题吗？

不是必须的。Skill会根据你的回答灵活调整：
- 如果某个选题不适合，会提前指出
- 如果素材不够，会重点追问
- 如果你已经有素材，可以跳过引导直接创作

### Q3：批判审查会不会太严格？

批判审查的目的不是"挑刺"，而是确保质量：
- **真实性**：确保内容经得起推敲
- **吸引力**：确保读者不会走神
- **信息密度**：确保没有废话
- **AI味**：确保像真人写的

如果你提供的素材本身就很好，审查会很顺利。如果发现问题，也会给出具体的修改建议。

### Q4：如何更新我的个人风格？

你的语言风格存储在 `style_persona.md` 中。

如果你的写作风格有变化，可以：
1. 提供新的文章样本
2. 让Skill重新学习
3. 更新 `style_persona.md`

### Q5：生成的文章能直接发布吗？

建议不要直接发布，而是：
1. 通读一遍，看是否符合你的预期
2. 补充一些只有你知道的细节
3. 调整一些表达，让它更像"你"
4. 配图、排版

把Skill当作你的**写作搭子**，而不是**替代品**。

## 最佳实践

### 1. 用于什么场景？

✅ **适合**：
- 公众号深度文章（1500-3000字）
- 需要个人观点和经历的内容
- 评测、方法论、行业观察类文章
- 需要"活人感"的内容

⚠️ **不太适合**：
- 新闻资讯（时效性太强，没时间引导）
- 纯理论科普（缺少个人经历）
- 短文案（字数太少，引导成本高）

### 2. 推荐工作流

```
1. 有了选题想法
   ↓
2. 使用Skill进入引导模式
   ↓
3. 回答9个问题（准备好素材）
   ↓
4. 确认大纲
   ↓
5. 分段写作 + 批判审查（逐段打磨）
   ↓
6. 全文审查（HKR检验）
   ↓
7. 导出文章，补充细节和配图
   ↓
8. 发布
```

### 3. 如何提升效果？

**在引导阶段**：
- 提供越具体的素材，文章质量越高
- 不要编造经历，真实的"不完美"比虚假的"完美"更有价值
- 大胆表达真实想法，哪怕主观或偏激

**在审查阶段**：
- 看到"信息密度低"的提示，狠心删
- 看到"AI味过重"的提示，大胆改
- 不要追求完美，允许一些"毛刺"

**发布后**：
- 收集读者反馈，看哪些部分最受欢迎
- 分析数据，看完读率、分享率
- 用这些反馈优化下次创作

## 技术原理

### 为什么这个Skill能降低AI味？

**传统AI写作的问题**：
1. 缺少真实素材（都是AI编的）
2. 语言风格通用化（没有个人特征）
3. 结构套路化（总-分-总）
4. 追求完美（没有真人写作的"毛刺"）

**这个Skill的解决方案**：
1. **引导挖掘真实素材** → AI编不出来的护城河
2. **学习个人语言风格** → 融入你的高频词汇和句式习惯
3. **批判审查机制** → 4个维度确保质量
4. **信息密度控制** → 拒绝废话和注水
5. **允许"不完美"** → 轻微重复、口语化、思路跳跃才是真人特征

### 融合的方法论

这个Skill融合了三大来源：
1. **原提示词**：去AI味技巧、结构设计
2. **卡兹克方法论**：活人感、HKR检验、选题三要素
3. **你的个人风格**：真实文章的语言特征提取
